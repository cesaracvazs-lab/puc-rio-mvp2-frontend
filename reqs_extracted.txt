Requisitos  e  Composição  da  Nota  
Pontuação  Requisito  
Componentização  (3,5  pts)  
0,5  Devem  ser  utilizados  os  princípios  de  componentização  para  dividir  sua  aplicação  em   3  páginas  em  componentes  reutilizáveis.  
2,0  Reutilizar  pelo  menos  4  componentes .  
Lista  de  sugestões:  Header  com  logo  e  links  de  navegação;  Campo  de  busca  com  input  e  botão;  Card  de  serviço  com  imagem,  título,  descrição  e  botão;  Botão  com  estilo  próprio  e  animações;  Menu  de  navegação  lateral  ou  superior;  Alerta  de  feedback  como  sucesso,  erro  ou  aviso;  Item  de  lista  ou  linha  de  tabela  reutilizável;  Tooltip  com  explicações  rápidas;  Modal  de  confirmação  para  ações  críticas.  
1,0  Apresentar  soluções,  funcionalidades  ou  interfaces  que  se  diferenciam  do  exemplo  base  fornecido.  É  importante  que  o  sistema  tenha  elementos  originais,  como  abordagens  alternativas  de  navegação,  componentes  interativos  personalizados  ou  propostas  visuais  distintas.  
 Observação:  Devem  ser  simuladas  as  requisições  a  um  servidor,  isso  é,  ao  invés  de  realizar  uma  requisição,  deve-se  fazer  a  leitura  de  um  JSON  com  o  dado  desejado.  
React  e  Roteamento  de  Páginas  (2,5  pts)  
1,0  Realizar  o  uso  correto  de  estados,  props  e  hooks  (como  useState ,  useEffect ).  
Obs:  É  permitido  utilizar  outras  bibliotecas/frameworks  como  Vue ,  desde  que  a  estrutura  siga  os  princípios  de  componentização.  
1,0  A  aplicação  deve  implementar  a  navegação  entre  páginas  utilizando  o  React  Router (ou  ferramenta  equivalente,  caso  esteja  utilizando  outro  framework  ou  biblioteca).   Além  disso,  devem  ser  utilizados  pelo  menos  três  hooks  (ou  funções  equivalentes)  relacionados  à  navegação,  como:   ●  useNavigate  (ex:  redirecionamento  de  rotas);  ●  useParams  (ex:  captura  de  parâmetros  da  URL);  ●  useLocation  (ex:  leitura  da  URL  atual).  
Atenção!  Caso  você  esteja  usando  uma  biblioteca  ou  framework  diferente  do  React  Router,  deve  utilizar  os  conceitos  equivalentes  àqueles  exigidos  acima .  
0,5  A  aplicação  deve  incluir  uma  rota  de  erro  para  URLs  inexistentes  (ex:  exibir  uma  página  404  not  found ).  
Usabilidade  (2  pts)  
1,5  Incluir  alguns  elementos  para  melhorar  a  experiência  do  usuário,  tais  como:   ●  Exibir  feedback  visual  após  ações  do  usuário,  como:  cliques,  envios  de  formulário,  carregamento  de  dados;  ●  Tooltips  explicativas  em  botões  ou  ícones;  ●  Mensagens  condicionais  contextuais,  como  “nenhum  item  encontrado”  ou  “erro  ao  carregar  dados”.  ●  Indicadores  de  carregamento,  sucesso  ou  erro;  
0,5  Desenvolver  um  layout  adaptável,  que  se  ajuste  adequadamente  a  diferentes  tamanhos  de  tela  (desktop,  tablet  e  celular).  Não  é  necessário  que  o  design  seja  perfeito  em  todos  os  dispositivos,  mas  o  conteúdo  deve  permanecer  legível  e  funcional.  
Organização  e  documentação  dos  códigos  (2  pts)  
0,5  Projeto  hospedado  publicamente  no  GitHub.  
0,5  O  seu  README.md  deve  conter  as  seguintes  informações:  Título  e  uma  breve  descrição  do  projeto,  Instruções  de  Instalação  (tais  como:  descrever  as  etapas  necessárias  para  que  os  usuários  possam  configurar  o  ambiente  local,  instalar  dependências,  comandos  de  inicialização,  etc).  Certifique-se  de  que  o  arquivo  ReadMe.md  seja  formatado  de  forma  clara  e  use  cabeçalhos,  listas  e  formatação  de  texto  para  tornar  a  documentação  fácil  de  ler.  
0,5  Organização  clara  da  estrutura  de  pastas  e  arquivos  no  projeto.  
0,5  Uso  adequado  de  convenções  de  nomenclatura  e  boas  práticas  (nomes  de  componentes,  arquivos,  etc.)   Atenção!  Caso  sejam  utilizados  como  base  os  exemplos  apresentados  em  aula,  serão  penalizadas  as  entregas  que  não  apresentarem  pelo  menos  50%  de  código  novo.
